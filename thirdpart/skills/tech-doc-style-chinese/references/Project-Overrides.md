# 当前项目覆盖规则

这份文件存放当前项目的专属约定，不属于通用中文技术文档写作规范。

仅在当前项目或明确要求沿用当前项目文档体系时读取本文件。

## 版本展示

- 面向人阅读的文案中，当前项目优先使用 `2.0` / `3.0`
- 真实实现路径保持 `/v2/...`、`/v3/...`
- 如果正文提到的是路径字面量，只在路径内保留 `v2` / `v3`

## 当前项目术语偏好

以下术语适用于当前项目文档，可按需保留：

- `GitLab`
- `GitHub`
- `Content-Type`
- `iOS`
- `ATC`

如果这些术语只是在当前项目里偶尔出现，不必把它们上升为通用规则。

## 当前项目文档结构偏好

### 文档中心首页

首屏文案优先回答三件事：

- 文档覆盖什么
- 适合谁
- 推荐从哪里开始读

### 平台介绍页

第一段优先说明：

- 这页给谁看
- 这页解决什么问题
- 建议接着读哪里

并避免与后续 `平台定位` 或 `能力概览` 重复。

### 解决方案页

当前项目推荐顺序：

1. 适用场景
2. 典型问题
3. 推荐方案
4. 推荐调用顺序
5. 实施建议
6. 能力边界

### 接口文档页

- 请求方法行进入代码块
- 参数说明尽量一列一义
- 错误码文案使用统一口径
